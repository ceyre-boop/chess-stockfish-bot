2. PHASE 1 — EXECUTION SIMULATOR + MINIMAL COST MODEL
----------------------------------------------------------------------

Goal:
Make PnL and ELO honest enough not to lie, without over-engineering.

Tasks:
- Create engine/execution_simulator.py
  Inputs:
    - action, target_size
    - mid price
    - LiquidityState, VolatilityState
    - symbol config (tick size, spread)
  Outputs:
    - actual fill price
    - filled size (partial fills allowed)
    - transaction cost
    - updated PositionState

- Execution model (v1):
  - Spread: bid = mid - spread/2, ask = mid + spread/2
  - Slippage: slippage = k * ATR * (size / liquidity_scale)
  - Costs: fixed commission + optional per-notional fee

- Integrate into RealDataTournament:
  - All trades must go through ExecutionSimulator
  - No mid-price fills in official mode

- Add execution_config.yaml:
  - spread per symbol
  - slippage coefficient
  - commission

Failure-aware notes:
- Do not over-engineer execution.
- Re-run brutal tournaments after integration.

----------------------------------------------------------------------
3. PHASE 2 — DATA INTEGRITY LAYER + BIAS TESTS
----------------------------------------------------------------------

Goal:
Ensure strict time-causality and prevent survivorship or lookahead bias.

Tasks:
- Create data/data_integrity.py:
  - verify no future timestamps in features
  - verify time-aligned joins
  - verify no "as of now" fields leak future info

- Create tiny hand-crafted datasets:
  - a few days around a macro event
  - manually define what data is visible at each timestamp

- Add tests:
  - tests/test_bias_time_causality.py
  - tests/test_bias_survivorship.py

- Enforce in official runs:
  - official tournaments require data integrity checks to pass

Failure-aware notes:
- Any bias failure is a red alert.
- Fix immediately before continuing.

----------------------------------------------------------------------
4. PHASE 3 — PORTFOLIO RISK MANAGER + BASIC GOVERNANCE (v1)
----------------------------------------------------------------------

Goal:
Add simple global risk controls without overcomplicating correlations.

Tasks:
- Create engine/portfolio_risk_manager.py:
  - Track:
    - total capital
    - per-symbol exposure
    - total notional exposure
    - realized/unrealized PnL
  - Enforce:
    - max exposure per symbol
    - max total notional
    - max daily loss (portfolio-wide)

- Create engine/governance.py:
  - Global kill switch:
    - if daily loss > threshold:
      - stop new entries
      - optionally flatten positions

- Integrate into RealDataTournament:
  - Before applying any action:
    - ask PortfolioRiskManager and Governance
    - override action if not allowed

Failure-aware notes:
- Avoid double-counting risk.
- Clear boundaries:
  - PolicyEngine: per-position risk
  - PortfolioRiskManager: cross-position risk
  - Governance: meta-level kill switches

----------------------------------------------------------------------
5. PHASE 4 — ENGINE HEALTH MONITOR (SIMPLE, RISK-OFFICER STYLE)
----------------------------------------------------------------------

Goal:
Make the engine self-aware without turning health into a hidden optimizer.

Tasks:
- Create engine/health_monitor.py:
  Inputs:
    - rolling PnL
    - rolling Sharpe
    - rolling drawdown
    - regime labels
    - expected performance bands
  Outputs:
    - health_status ∈ {HEALTHY, DEGRADED, CRITICAL}
    - risk_multiplier ∈ {1.0, 0.5, 0.0}

- Define rules BEFORE seeing results:
  - If 3-month Sharpe < 0 in a regime → DEGRADED
  - If drawdown > threshold → CRITICAL

- Integrate:
  - PolicyEngine scales max_position_size by risk_multiplier

Failure-aware notes:
- Health rules must not be tuned to maximize PnL.
- Treat health monitor like a risk officer, not a strategist.
